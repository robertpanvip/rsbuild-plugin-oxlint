# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`. The contents of this folder are how changesets are stored.

## What are changesets?

A changeset is a piece of information about a change to your code. It describes:

- Which packages have changed
- What version the change should be released as
- A description of the change (for the changelog)

## How to use changesets

### 1. Make your changes

Make your code changes as usual.

### 2. Create a changeset

Run the following command:

```bash
pnpm changeset
```

This will prompt you to:

1. Select which packages have changed
2. Choose the version bump type (major, minor, or patch)
3. Write a description of the change

### 3. Commit the changeset

Commit the generated changeset file along with your code changes.

### 4. Release

When you push to the main branch, the GitHub Action will:

1. Create a "Version Packages" PR that updates versions and changelogs
2. When that PR is merged, it will publish the changed packages to npm

## Example

If you fixed a bug in `rsbuild-plugin-oxlint`:

```bash
pnpm changeset
```

Select:
- `rsbuild-plugin-oxlint`
- `patch` (for bug fixes)
- Description: "Fix issue with config file path resolution"

This creates a file like `.changeset/cool-feature-123.md`:

```md
---
"rsbuild-plugin-oxlint": patch
---

Fix issue with config file path resolution
```
